# Trace 2
This project is a high-precision input automation tool that records and replays keyboard and mouse events on Windows. It combines Python for orchestration and C for performance-critical replay execution.

The system captures user input in real time, stores it in a compact binary format, and then generates a standalone C program that replays the recorded events with accurate timing. This generated program is compiled into an executable, allowing deterministic and efficient playback without requiring the original Python environment.

## Project Structure

- `src/c/trace_input_recorder.c`: standalone low-level input recorder (keyboard + mouse)
- `src/python/trace_controller.py`: hotkey controller for record, compile, and replay
- `data/trace_events.bin`: binary trace output produced by recorder
- `generated/trace_output_static.c`: static replay source generated by controller
- `bin/`: compiled recorder/replay executables
- `legacy/`: old or experimental files preserved for reference
- `.vscode/`: local editor settings

## Quick Start

1. Install Python dependency:
   - `pip install pynput`
2. Make sure `gcc` is available in PATH.
   - `trace_controller.py` also tries common MSYS2 compiler paths such as `C:\msys64\ucrt64\bin` automatically.
3. Run controller:
   - `python src/python/trace_controller.py`

## Hotkeys

- `F4`: start recording
- `F5`: stop recording
- `F6`: compile recorder + static replay output
- `F2`: start replay
- `F3`: stop replay
- `F7`: toggle mouse replay on/off
- `F8`: toggle loop on/off

## Notes

- Replay uses `SendInput` with keyboard scancodes and mouse events.
- Precision is improved with QPC timestamping, waitable timer, and short spin-wait.
- Absolute precision is still constrained by Windows scheduler, CPU load, and game frame timing.
- Mouse replay can still show slight coordinate mismatch (for example a small right shift) depending on DPI scaling and display setup.
